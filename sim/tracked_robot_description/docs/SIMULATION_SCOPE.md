## 必须模拟

这些内容会直接影响算法、控制或传感器验证：

- 履带底盘运动：用 skid-steer / diff-drive 近似左右履带。
- 底盘碰撞体：用简化箱体和履带外轮廓，保证能验证过障、避障、碰撞检测。
- N10P 雷达：发布 `/scan`，位置和高度按实车。
- D435：发布 RGB / depth / camera_info，位置和高度按实车。
- 五舵机机械臂：ID1-ID4 做关节运动学，ID5 做夹爪开合。
- TF 树：`base_footprint -> chassis -> sensors -> arm links` 必须正确。

## 建议只做外观，不做真实物理

这些看起来像实车即可，初期不建议做复杂动力学：

- 履带齿、履带链节。
- 主动轮和下方 4 个承重轮的真实接触传动。
- 弹簧/减震器真实动力学。
- 电控板、线束、电池、螺丝、支架孔位。
- 机械臂双平行杆的精细连杆机构。

## 当前实现策略

- 外观：画出前后履带轮、下方 4 个承重轮、减震支柱、车体、雷达、D435、机械臂。
- 运动：Gazebo 使用隐藏等效履带轮驱动，避免真实履带接触模型不稳定。
- 机械臂：先用简化串联关节模型验证工作空间和控制接口，外观用双平行管近似实车。
- 展示：如果后续需要更高视觉保真度，再替换 CAD/Blender mesh。

## 什么时候需要高精模型

只有这些场景才值得继续细化：

- 需要仿真越障、爬坡、履带打滑。
- 需要机械臂碰撞避障很精确。
- 需要和真实相机画面做高保真域随机化。
- 需要对外展示，且要求视觉上接近实车。

当前阶段不需要高精履带物理模型。先保证 Gazebo 能稳定运行、传感器 topic 正常、机械臂 TF 和关节控制正确。
